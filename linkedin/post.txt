🚀 DevSecOps + GitOps Pipeline: TechStore Project

I recently built TechStore, a full-stack e-commerce application deployed using a robust DevSecOps CI/CD pipeline with GitOps on AWS EKS.

The goal of this project was to implement a production-grade automated pipeline — from code commit to a secure, monitored Kubernetes deployment, following modern Cloud Engineering practices.

🔁 Project Flow

1. Code Commit: It starts when I push code to GitHub. 
2. Continuous Integration: This triggers a Jenkins pipeline that automates the entire lifecycle.
3. Security & Quality Gates:
    * SonarQube performs Static Analysis (SAST) to ensure high code quality.
    * Trivy scans both the filesystem and the final container images for vulnerabilities.
4. Containerization: Once validated, the app is containerized using Docker and pushed to Docker Hub.
5. GitOps Deployment: The pipeline automatically updates the Kubernetes Manifests with the new image tag. 
6. CD with ArgoCD: ArgoCD detects the repository update and automatically synchronizes the state with the AWS EKS cluster.
7. Observability: Prometheus & Grafana provide real-time monitoring of the cluster and application health.
8. Automated Feedback: Constant status updates are sent via Email Notifications for every success or failure in the pipeline.

🚧 Challenges Overcome

* Pipeline Orchestration: Correctly sequencing SonarQube quality gates to stop the build if security standards weren't met.
* GitOps Sync: Ensuring ArgoCD was correctly configured to watch the manifest repository and handle rolling updates without downtime on EKS.
* Secrets Management: Managing security credentials for Docker Hub and GitHub within Jenkins securely using environment credentials.

🏆 Key Learnings & Successes (The STAR Method)

🔹 Situation: Ensuring 100% secure deployments on a production-grade Kubernetes cluster (EKS).
🔹 Task: Integrating automated security gates without slowing down the development lifecycle.
🔹 Action: Implemented automated Quality Gates in SonarQube and sequential Trivy scans. I also automated the GitOps tag updates using custom 'sed' scripts in the Jenkins pipeline to ensure a hands-off deployment loop.
🔹 Result: A fully automated, zero-trust pipeline where security is enforced 100% of the time, and deployments are consistent and reproducible via ArgoCD.

🧰 Tech Stack

AWS (EKS, EC2, VPC) • Jenkins • Docker & Docker Hub • SonarQube • Trivy • ArgoCD • Kubernetes • Prometheus & Grafana • Node.js (Express) • HTML/CSS/JS • GitHub
